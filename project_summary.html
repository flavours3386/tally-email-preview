아임웹 프로모션 신청 자동화 시스템

프로젝트 요약 — 문제 정의부터 해결까지

1. 배경: 어떤 문제가 있었나

알파리뷰 x 아임웹 런칭 프로모션을 시작하면서, Tally 폼으로 신청을 받기 시작했습니다.

문제 1 — 신청 감지가 느림
Tally 폼 제출 후 Google Sheets에 데이터가 쌓이지만, 담당자가 시트를 직접 확인하기 전까지는 신청이 들어온 줄 알 수 없었습니다. 프로모션 특성상 빠른 응대가 중요한데, 신청 후 수 시간이 지나서야 확인하는 경우가 발생했습니다.
문제 2 — 수동 반복 업무
신청자 정보를 하나하나 확인하고, Stibee 주소록에 수동 등록하고, 환영 메일을 직접 보내야 했습니다. 신청 건수가 늘어날수록 이 작업에 소요되는 시간도 비례해서 증가했습니다.
문제 3 — 설치 전환율 관리 부재
프로모션 신청 후 실제 앱 설치까지 이어지는지 체계적으로 추적하거나, 미설치 신청자에게 리마인드를 보내는 프로세스가 없었습니다.

2. 해결: 무엇을 만들었나

Tally 폼 제출부터 이메일 시퀀스까지 전 과정을 자동화하는 시스템을 구축했습니다.

3. 과정: 어떤 문제를 만나고, 어떻게 풀었나

Phase 1 — 내부 알림

Google Chat 웹훅 + Apps Script로 새 신청 감지 시 담당자에게 즉시 알림.
onChange 트리거 + 1분 폴링 2중화로 누락 방지.
해결 완료

Phase 2 — Stibee 연동

신청 접수 시 Stibee 주소록에 자동 등록하고, 프로모션 유형(VIP/얼리아답터/일반)에 맞는 그룹에 자동 할당.
설치완료 시 프로모션 그룹에서 제거하고 설치완료 그룹으로 자동 이동.
해결 완료

Phase 2-2 — 이메일 시퀀스

이 단계에서 가장 많은 시행착오가 있었습니다.

시도 1: Stibee 자동 메일 불가
Stibee의 자동 메일 기능은 프로 요금제 전용. 현재 스탠다드 플랜이라 1통도 발송 불가.
시도 2: Stibee 1차 + Apps Script 나머지 불가
1차 메일도 Stibee 자동 메일이 필요해서 동일한 이유로 불가.
시도 3: Pipedrive 시퀀스 연동 불가
시퀀스 API 미제공 (UI 전용). Automations로 시퀀스 추가는 되지만, 설치완료 시 시퀀스에서 제거하는 방법이 없음.
최종 해결: 전체 Apps Script + GmailApp 채택
3통 모두 GmailApp으로 HTML 디자인 메일 발송. 설치완료 시 발송 스킵이 간단하고, Stibee는 구독자 관리용으로 유지. 추가로 Stibee 수신거부 상태를 API로 확인하여 수신거부자에게는 발송하지 않도록 처리.

7. 기대 효과

  • 응대 속도 개선 — 신청 후 1-2분 내 담당자 알림 + 고객에게 즉시 환영 메일
  • 반복 업무 제거 — 구독자 등록, 메일 발송, 설치 추적 전량 자동화
  • 설치 전환율 개선 — 3통 시퀀스로 지속적 설치 유도, 미설치자에게 자동 리마인드
  • 프로모션 유형별 맞춤 커뮤니케이션 — VIP는 전담 매니저 톤, 일반은 셀프서브 톤으로 차별화
  • 운영 비용 최소화 — 추가 유료 도구 없이 기존 Google Workspace + Stibee 스탠다드 내에서 구현
