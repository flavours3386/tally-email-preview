# Tally 이메일 미리보기 - Product Sense

## 미션

프로모션 이메일 시퀀스의 실제 모습을 누구나 브라우저에서 확인할 수 있게 하고, 운영 가이드를 통해 자동화 시스템의 가치를 비개발자에게 전달한다.

## 해결하는 문제

- Apps Script에서 HTML 메일을 코드로 작성하면 실제 발송 전까지 렌더링 결과를 확인할 수 없음
- 마케팅팀이 이메일 톤앤매너를 리뷰하려면 코드를 읽어야 함 (비효율적)
- 자동화 시스템의 전체 흐름을 경영진/협업팀에게 설명할 때 별도 문서가 필요

## 사용자

| 사용자 | 역할 | 사용 시나리오 |
|--------|------|-------------|
| 마케팅팀 | 이메일 리뷰 | 프로모션 유형별 이메일 톤앤매너/CTA/배너 확인 및 피드백 |
| 세일즈팀 | 이메일 내용 확인 | 고객에게 어떤 메일이 가는지 파악. VIP 전담 톤 검수 |
| CSM팀 | 운영 흐름 확인 | 설치완료 시 시퀀스 중지 등 자신이 담당하는 부분의 동작 확인 |
| 경영진 | 프로젝트 이해 | operations_guide.html로 시스템 전/후 비교, 비용 효과 파악 |
| 개발자 (본인) | 이메일 디버깅 | HTML 렌더링 결과를 브라우저에서 빠르게 확인 |

## 핵심 가치

1. **비개발자 접근성**: URL 하나로 이메일 미리보기/운영 가이드에 접근. 코드를 몰라도 리뷰 가능.
2. **프로모션 유형별 비교**: VIP/얼리아답터/일반 3벌의 차이점을 한눈에 비교 가능.
3. **공유 가능한 문서**: Netlify URL을 슬랙/메일로 공유하여 비동기 피드백 수집.

## 제품 원칙

- 이 사이트는 "보기 전용"이다. 편집 기능은 제공하지 않는다.
- 미리보기 HTML은 실제 발송 HTML과 가능한 한 동일해야 한다 (WYSIWYG).
- 운영 가이드는 기술 용어를 최소화하고 전/후 비교를 중심으로 구성한다.

## 성공 지표

|------|----------|------|
| 이메일 리뷰 주기 | 마케팅팀 피드백 반영 빈도 | 이메일 변경 시마다 사전 리뷰 |

## 경쟁 대안

| 대안 | 장점 | 이 방식을 선택한 이유 |
|------|------|---------------------|
| 테스트 메일 발송 | 실제 이메일 클라이언트에서 확인 | 발송 한도 소모 + 3종 x 3통 = 9통씩 발송해야 함 |
| Stibee 미리보기 | 대시보드에서 바로 확인 | 현재 GmailApp으로 발송하므로 Stibee에 템플릿이 없음 |
| 이메일 테스트 도구 (Litmus 등) | 다양한 클라이언트 렌더링 확인 | 유료. 현재 규모에서는 과잉 |
| 정적 HTML + Netlify (현재) | 무료, URL 공유, 즉시 배포 | 규모에 적합. 빌드 없이 HTML만 push하면 배포 완료 |
