# Tally 이메일 미리보기 - Plans

> 최종 업데이트: 2026-04-02

## 현재 우선순위

1. **Tally_webhook과 동기화 확인** — buildEmailHtml(), getMail*Content() 변경사항이 미리보기 HTML에 반영되어 있는지 확인.
2. **Netlify 배포 상태 확인** — https://stirring-sprinkles-29cf5a.netlify.app/ 접속하여 최신 버전 배포 확인.
3. **외부 이미지 URL 유효성** — 배너 이미지, 데모 GIF의 Stibee CDN URL이 정상 로드되는지 확인.

## 로드맵

### Phase 1 (완료): 미리보기 + 운영 가이드
- 프로모션 유형별 이메일 미리보기 HTML 3벌 (VIP/얼리아답터/일반)
- 각 파일에 메일 1/2/3 순서대로 나열 (제목/시점 라벨 포함)
- 운영 가이드 HTML (비개발자 대상 자동화 시스템 소개)
- 프로젝트 요약 HTML (기술 구현 포인트)
- Netlify 정적 배포

### Phase 2 (미착수): 개선
- 인덱스 페이지: 모든 미리보기/가이드를 한곳에서 탐색할 수 있는 랜딩 페이지
- 자동 동기화: Tally_webhook의 gs 코드에서 HTML을 자동 추출하여 미리보기 생성
- 다크모드 이메일 미리보기: 다크모드 이메일 클라이언트에서의 렌더링 시뮬레이션

## 기술적 의사결정

### 왜 정적 HTML인가?
- 빌드 도구/프레임워크 없이 HTML 파일만 push하면 배포 완료
- Netlify 무료 플랜으로 충분 (정적 사이트)
- 이메일 HTML 자체가 인라인 스타일 + 테이블 레이아웃이므로 변환 없이 그대로 미리보기

### 왜 프로모션 유형별 파일을 분리하는가?
- VIP/얼리아답터/일반은 톤(전담 매니저 vs 셀프서브)과 CTA가 다름
- 한 파일에 9개 메일을 넣으면 스크롤이 너무 길어짐
- 마케팅팀 리뷰 시 유형별로 URL을 별도 공유할 수 있어 편리

### 왜 Netlify인가?
- GitHub 레포 연결만으로 자동 배포 (CI/CD 설정 불필요)
- 커스텀 도메인 없이 기본 URL로 충분
- 무료 플랜 대역폭이 월 100GB (이 규모에 충분)
