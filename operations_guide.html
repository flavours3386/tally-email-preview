1. 시스템 개요

Tally 폼으로 접수된 프로모션 신청을 자동으로 처리하는 시스템입니다.

Tally 폼 제출 → Google Sheets 적재 → Apps Script 감지 → Google Chat 알림 + Stibee 등록 + 이메일 발송

3. 이메일 시퀀스

프로모션 신청자에게 자동으로 3통의 이메일을 발송합니다.

순서발송 시점목적CTA
메일 1신청 직후 즉시 (~1-2분)접수 확인 + 설치 안내알파리뷰 설치하기
메일 2신청일 +3일, 오전 10시데모 사이트로 가치 전달데모 사이트 보기
메일 3신청일 +7일, 오전 10시도입/체험 도움 제안도입 가이드 보기 (일반/얼리)
알파리뷰 설치하기 (VIP)

프로모션 유형별 차이

VIP얼리아답터 / 일반
메일 1전담 매니저 안내 톤셀프 설치 안내 톤
메일 2"전담 매니저가 세팅 도와드릴게요""설치 가이드 따라하시면 1분이면 완료"
메일 3"전담 매니저가 처음부터 함께""편하게 회신 주시면 바로 안내"

발송 규칙

  • 설치여부가 "설치완료"이면 남은 메일 전체 스킵
  • 메일 2, 3 발송 전 Stibee 수신거부 상태 확인 → 수신거부 시 스킵
  • 이전 메일이 미발송이면 다음 메일도 발송하지 않음 (순서 보장)
  • 한 트리거 실행당 한 행에서 한 통만 발송
  • 발송 완료 시 해당 컬럼에 타임스탬프 자동 기록

이메일 레이아웃

  • 상단: 안내 문구 ("본 메일은 아임웹X알파리뷰 프로모션 신청자에게 발송됩니다.")
  • 배너: 알파리뷰 이미지
  • 본문: 인사말 + 내용 (메일 2에는 데모 GIF 포함)
  • 하단: 풀너비 CTA 버튼 (#837AFF)
  • 푸터: 회사 정보 + 수신거부 링크

4. 트리거 구성

트리거함수주기역할
onChangeprocessNewSubmissions시트 변경 시새 신청 감지 → Chat 알림 + Stibee 등록 + 메일 1 발송
시간 기반processNewSubmissions1분마다onChange 누락 대비 폴링
시간 기반sendSequenceEmails매일 오전 10시메일 2~3 경과일 체크 후 발송
onEditonEditHandler셀 편집 시설치여부 "설치완료" 감지 → Stibee 그룹 이동
트리거 설정은 Apps Script 에디터에서 setupTriggers 함수를 실행하면 자동 등록됩니다.
두 시트(얼리아답터/일반, VIP) 모두 각각 실행해야 합니다.

5. 일상 운영

평소에 할 일

기본적으로 모든 처리가 자동입니다. 별도 작업이 필요 없습니다.

설치완료 처리

  1. 해당 신청자 행의 설치여부 컬럼에서 "설치완료" 선택
  2. 자동 처리: Stibee 프로모션 그룹 제거 → 설치완료 그룹 이동, 남은 이메일 스킵

수동 발송 (필요 시)

시트 상단 메뉴에서:

  • Tally 알림 → 미발송 건 알림 보내기: 누락된 Chat 알림 + Stibee 등록 재처리
  • Tally 알림 → 이메일 시퀀스 수동 발송: 미발송 이메일 즉시 발송

8. 트러블슈팅

증상원인해결
새 신청인데 알림이 안 옴 트리거 만료 또는 오류 Apps Script에서 setupTriggers 재실행
이메일이 발송되지 않음 Gmail 일일 발송 한도 초과 또는 권한 문제 Apps Script 실행 로그 확인 (실행 → 실행 기록)
메일 1만 발송되고 2, 3이 안 감 경과일 미충족 또는 설치완료 표시됨 Submitted at 기준 +3일/+7일 대기, 설치여부 컬럼 확인
Stibee 등록 실패 API 키 만료 또는 주소록 ID 변경 스크립트의 API 키 및 ID 값 확인
설치완료 처리가 안 됨 드롭다운 값이 정확히 "설치완료"가 아님 셀 값이 공백 없이 "설치완료"인지 확인
중복 알림/메일 발송 일반적으로 발생하지 않음 "알림발송", "메일N발송" 컬럼에 값이 있는지 확인
